# Sprint 4 (недели 7-8): Pilot Packaging

## Цель спринта

Подготовить решение к продаже пилота: интеграционный API v1, продуктовый демо-пакет и коммерческие документы.

## Объем работ

- API v1 для внешней интеграции (REST/gRPC, минимальный контракт)
- Адаптер-заглушка под MES/ERP
- Productized demo (сценарий + метрики + отчет; **запись демо из Webots** при необходимости для клиента)
- Draft документов Pilot SOW/SLA

## Технические задачи

- Зафиксировать минимальный API-контракт и версионирование
- Реализовать базовые endpoint/methods и примеры вызовов
- Подготовить mock-адаптер внешней системы (MES/ERP stub)
- Упаковать demo-стенд с воспроизводимыми KPI-отчетами

## Коммерческие задачи

- Подготовить draft `SOW` (scope, таймлайн, критерии приемки)
- Подготовить draft `SLA` (операционные допущения и KPI)
- Синхронизировать KPI пилота с клиентским контекстом

## Артефакт спринта

- Пакет для продажи пилота (`Pilot-as-a-Product package`)

## Definition of Done

- API v1 описан и работает на базовом сценарии
- MES/ERP stub демонстрирует интеграционный путь
- Productized demo готов к показу клиенту
- Черновики SOW/SLA согласованы внутри команды

## KPI контроля

- Время интеграции через API v1 (time-to-first-mission)
- Доля успешных интеграционных вызовов
- Полнота пилотного пакета (чеклист готовности)

## Зависимости

- Reliability baseline из Sprint 3
- Подтвержденный целевой use-case клиента

## Риски и снижение

- Риск: over-engineering API на раннем этапе  
  Митигирование: минимальный контракт + явный out-of-scope
- Риск: несоответствие SOW реальному runtime-поведению  
  Митигирование: привязка обещаний к измеряемым KPI
