# Sprint 6 (недели 11-12): Pre-pilot Hardening II

## Цель спринта

Довести observability и интеграционную готовность до уровня `go-live readiness` для первого пилота.

## Объем работ

- Hardening observability (алерты, инцидент-таймлайн)
- Подготовка перехода на 1 реальный робот-адаптер
- Финальный go-live readiness checklist

## Технические задачи

- Ввести алертинг на ключевые reliability-сигналы
- Реализовать инцидент-таймлайн для разбора событий миссии
- Подготовить adapter contract для 1 реального робота
- Выполнить pre-flight проверку совместимости sim (**Webots/PyBullet-контур**) -> real adapter

## Артефакт спринта

- `Go-live readiness checklist` + итоговый pre-pilot hardening report

## Definition of Done

- Доступны алерты по критичным отказам и деградациям
- По инциденту собирается цельный таймлайн событий
- Подготовлен план включения 1 реального адаптера
- Команда имеет формальный go/no-go чеклист к старту пилота

## KPI контроля

- MTTD/MTTA по критичным инцидентам
- Полнота событий в инцидент-таймлайне
- Процент выполненных пунктов go-live checklist

## Зависимости

- Hardening I из Sprint 5
- Согласованный формат интеграции с pilot-контуром клиента

## Риски и снижение

- Риск: расхождение поведения sim и реального контура  
  Митигирование: adapter contract tests + staged rollout
